Project Description

Overview

  • In this assignment, I was tasked to redesign an online scheduling system and collect data to compare my modified design against the original. To do this, I utilized hypothesis testing and drawings from statistically significant analysis.
  • We ran an A/B test on two different UI designs and analyzed the results from our findings.
  • We used the results to draw conclusions on the effects of alternate design choice and the impact it has on a user's interaction
  • We performed the target interaction on a class designated machine (version A), modified version B to test a design change from version A, and finally we performed target interactions on ~30 different version B's.

What is A/B Testing?

  • A/B testing is responisble for comparing two versions of a product via user testing to figure out which one preforms better.
  • A refers to the control/original interface. Whereas B refers to the variation of the original platform.

What is the overarching question you are attempting to prove/disprove?

  • We are attempting to prove how visual changes to a design impacts metrics such as misclick rates, time spent on a page, and more.

How is A/B Testing useful?

  • A/B testing allows engineers to critically analyze data-driven decisions about changes made to a design.
  • Allows us to collect quantitative data to compare between the two variations.
  • A/B is an opportunity to collect user feedback to further enhance the user experience.
  • We can extrapolate meaningful insights from our results, drawing conclusions based on statistically significant data.
